# 湖北大学流浪动物救助信息管理系统数据库操作文档

## 1. 数据库设计

### 1.1 数据库类型选择
- **数据库类型**：MySQL
- **版本建议**：5.7+或8.0+
- **选择理由**：MySQL是一种关系型数据库，适合存储结构化数据，支持复杂查询和事务处理，性能和可靠性都比较好，适合本系统的业务需求。

### 1.2 数据库架构
- **数据库名称**：`hubu_animal_rescue`
- **字符集**：`utf8mb4`
- **排序规则**：`utf8mb4_unicode_ci`

## 2. 数据库操作

### 2.1 数据库创建
1. 登录MySQL：
   ```bash
   mysql -u root -p
   ```

2. 创建数据库：
   ```sql
   CREATE DATABASE IF NOT EXISTS hubu_animal_rescue DEFAULT CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
   ```

3. 使用数据库：
   ```sql
   USE hubu_animal_rescue;
   ```

### 2.2 表结构创建
运行`database.sql`文件创建表结构：
```bash
mysql -u root -p hubu_animal_rescue < database.sql
```

### 2.3 数据备份
运行`backup/backup.sql`文件备份数据：
```bash
mysql -u root -p hubu_animal_rescue < backup/backup.sql
```

### 2.4 数据迁移
运行`backup/migrate.sql`文件进行数据迁移：
```bash
mysql -u root -p hubu_animal_rescue < backup/migrate.sql
```

## 3. 后端服务配置

### 3.1 环境要求
- Node.js 14+
- npm 6+

### 3.2 依赖安装
```bash
npm install
```

### 3.3 数据库连接配置
修改`config/db.js`文件中的数据库连接参数：
```javascript
const pool = mysql.createPool({
  host: 'localhost',
  user: 'root', // 请根据实际情况修改
  password: 'root', // 请根据实际情况修改
  database: 'hubu_animal_rescue',
  waitForConnections: true,
  connectionLimit: 10,
  queueLimit: 0
});
```

### 3.4 启动服务
```bash
npm start
```

## 4. API接口

### 4.1 动物相关接口
- `GET /api/animals` - 获取所有动物
- `GET /api/animals/:id` - 根据ID获取动物
- `POST /api/animals` - 添加动物
- `PUT /api/animals/:id` - 更新动物
- `DELETE /api/animals/:id` - 删除动物
